# Isolate and reset demo sessions

Each Demo Mode session receives an isolated copy of the five Demo Sellers, 25 Demo Listings, and their transaction state. Actions taken by one judge never affect another judge or protected builder accounts. A confirmation-protected Reset Demo control restores only that session's demo data to its initial state, ensuring every judge begins with a complete and working marketplace.

Demo Mode also provides clearly labelled, session-only time controls that advance to the five-minute Checkout Hold expiry, two-hour proposal deadline, six-hour agreement deadline, 48-hour handover limit, cancellation boundary, or 15-minute no-show grace period. Normal prototype use follows the actual clock and displays in-app countdowns and notifications; external email, SMS, browser push, and background delivery are deferred.

Demo Mode provides equivalent session-only location controls for setting each sample party as away, near the Handover Point, or inside the 100 m confirmation area, together with simulated accuracy. Handover actions activate only when both sample parties satisfy the same schedule and location conditions as real accounts. The interface states that location supports the claim but does not prove a face-to-face meeting; real accounts use device location and never see simulation controls.
