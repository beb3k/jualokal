# Cap listing discovery at ten kilometres

Jualokal never shows a buyer any listings more than 10 km from the buyer's Browsing Location because a wider market would undermine its near-home purpose and recreate the travel burden it is designed to remove. The deliberately narrow prototype uses a fixed 2 km Discovery Radius; Jualokal may configure a different fixed radius for a future market, but buyers do not control it and it must never exceed 10 km. Exact Home Anchors, Seller Convenience Zone boundaries, and Pilot Area boundaries remain hidden. The prototype derives only **Under 1 km** and **1-2 km** Distance Bands privately from the protected Home Anchor and may show a Seller Discovery Marker as approximate area information. Panning changes only the visible viewport, while Recenter returns the camera to the stored Browsing Location; neither requests a new location nor changes discovery eligibility, and the prototype does not support remote-area search.

Viewing listings requires a Browsing Location obtained once on marketplace entry. **Refresh nearby listings** explicitly requests a new snapshot, discards the previous snapshot, recalculates Discoverable Listings and Distance Bands, and recenters the map while preserving the selected Discovery View and Category Filter. When the app resumes with a Browsing Location considered stale, it prompts the member to refresh rather than updating silently. Jualokal does not continuously track location, show the snapshot to Sellers, or retain it as location history. If location permission is denied or location is unavailable, the member sees an explanation instead of listings; Demo Mode follows the same explicit model with simulated snapshots.
